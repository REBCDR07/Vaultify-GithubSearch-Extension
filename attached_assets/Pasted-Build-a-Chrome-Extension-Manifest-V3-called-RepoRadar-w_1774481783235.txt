Build a Chrome Extension (Manifest V3) called "RepoRadar" with the following complete file structure. Generate ALL files at once, production-ready, no placeholders.

## STACK
- Manifest V3, vanilla JS ES Modules, no build step, no npm
- Background Service Worker for all API calls
- chrome.storage.local for all persistence
- Dark mode only UI

extension/

## UTILS/STORAGE.JS
chrome.storage.local wrappers:
- storageGet(key), storageSet(key, value), storageRemove(key), storageClear()
- saveGroqKey(key) / getGroqKey() — store as btoa(key)
- saveGithubToken(token) / getGithubToken() — store as btoa(token)
- getFavorites() / saveFavorite(fav) / removeFavorite(repoFullName) / isFavorite(name)
- getFolders() / saveFolder(folder) / deleteFolder(folderId)
- getCached(queryKey) / setCache(queryKey, data) — TTL 30 minutes
- getPrefs() / savePrefs(prefs) — default: {language:'fr', resultsPerPage:10, model:'llama-3.3-70b-versatile'}
- getStorageUsage() — returns bytes used
- addSearchHistory(entry) / getSearchHistory()

Favorite schema: {id, folderId, repoFullName, repoUrl, repoStars, repoLanguage, repoTopics[], aiSummary, personalNote, tags[], savedAt}
Folder schema: {id, name, icon (emoji), createdAt}

## UTILS/GROQ.JS
GROQ_MODELS array:
- {id:'llama-3.3-70b-versatile', name:'Llama 3.3 70B Versatile', speed:'~280 t/s', status:'recommended'}
- {id:'llama-3.1-8b-instant', name:'Llama 3.1 8B Instant', speed:'~560 t/s', status:'stable'}
- {id:'meta-llama/llama-4-maverick-17b-128e-instruct', name:'Llama 4 Maverick 17B', speed:'—', status:'preview'}
- {id:'meta-llama/llama-4-scout-17b-16e-instruct', name:'Llama 4 Scout 17B', speed:'—', status:'preview'}
- {id:'qwen-qwq-32b', name:'Qwen 3 32B', speed:'—', status:'preview'}
- {id:'moonshotai/kimi-k2-instruct', name:'Kimi K2', speed:'—', status:'preview'}

Functions (all async, all return {data, usage}):
1. groqChat({apiKey, model, messages, temperature, maxTokens}) → fetch POST https://api.groq.com/openai/v1/chat/completions
2. reformulateQuery({apiKey, model, userQuery, filters}) → returns {queries:string[3], suggestions:string[4], usage}
   System prompt: expert GitHub Search API, return ONLY JSON {"queries":[],"suggestions":[]}, use qualifiers language:, stars:>, pushed:>, topic:, in:readme
3. scoreRepositories({apiKey, model, userQuery, repos}) → returns {indices:number[], usage}
   System prompt: return ONLY JSON array of indices, max 10, ordered by relevance+maintenance+popularity
4. generateSummaries({apiKey, model, userQuery, repos}) → returns {summaries:{[repoFullName]:{}}, usage}
   Each summary: {name, what, usecases, stack, strengths} all under 80 chars

## UTILS/GITHUB.JS
- searchRepositories({query, token, perPage, page}) → fetch https://api.github.com/search/repositories
  Headers: Accept vnd.github+json, X-GitHub-Api-Version 2022-11-28, Authorization Bearer {token}
  Handle 403 rate limit and 401 invalid token with clear French error messages
- parallelSearch({queries:string[], token}) → run 3 queries simultaneously with Promise.allSettled, deduplicate by repo id
- formatRepo(repo) → clean object with: id, full_name, name, owner, description, url, stars, forks, language, topics[], license, pushed_at, pushedAgo (human: "il y a 3j"), starsFormatted ("4.2k")

## BACKGROUND/SERVICE-WORKER.JS
chrome.runtime.onMessage listener handling:
- SEARCH: full 5-step pipeline:
  1. reformulateQuery (Groq)
  2. parallelSearch (GitHub ×3)
  3. scoreRepositories (Groq, top 10 from 60-90 results)
  4. generateSummaries (Groq, top 10)
  5. return {repos[], suggestions[], queries[], rateLimit, totalUsage, fromCache}
  Cache with getCached/setCache, TTL 30min
- VALIDATE_GROQ_KEY: GET https://api.groq.com/openai/v1/models → {valid, error?}
- VALIDATE_GITHUB_TOKEN: GET https://api.github.com/user → {valid, login?, error?}
- GET_STATUS: returns {hasGroqKey, hasGithubToken, ready}
chrome.runtime.onInstalled: open options page with ?onboarding=true on first install
