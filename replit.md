### 2. RepoRadar (Chrome Extension)
Located in `extension/` — Manifest V3, vanilla JS ES Modules, no build step.

#### File Structure
```
extension/
├── manifest.json              — MV3 config, permissions: storage, identity, tabs
├── background/
│   └── service-worker.js      — All API calls (Groq + GitHub), search pipeline
├── popup/
│   ├── popup.html/css/js      — 420×600px dark UI, search + results
├── options/
│   ├── options.html/css/js    — Full settings page (API keys, model, prefs, data)
├── favorites/
│   ├── favorites.html/css/js  — Full favorites manager with folders
├── utils/
│   ├── storage.js             — chrome.storage.local wrappers, keys, favorites, cache
│   ├── groq.js                — Groq API: reformulate, score, summarize repos
│   └── github.js              — GitHub Search API, parallel search, formatRepo
└── assets/icons/              — icon16/32/48/128.png (generated by generate_icons.py)
```

#### Design System
- Colors: bg #0d1117, surface #161b22, border #30363d, blue #58a6ff, green #3fb950, orange #e3b341, red #f78166
- Fonts: Rajdhani (body), Unbounded (brand), Oswald (labels/UI)

#### Loading in Chrome
1. Go to `chrome://extensions`
2. Enable "Developer mode"
3. Click "Load unpacked" → select the `extension/` folder

#### Search Pipeline (5 steps in service worker)
1. Groq reformulates query into 3 GitHub search queries
2. GitHub parallel search (3 simultaneous queries, deduplicated)
3. Groq scores top 10 repos by relevance/maintenance/popularity
4. Groq generates AI summaries per repo
5. Cache result for 30 minutes

#### Key Security Practices
- API keys stored obfuscated (btoa) in chrome.storage.local, never localStorage
- Keys never logged to console
- All API calls happen in service worker only (never popup)
- CSP: `script-src 'self'; object-src 'self'`
